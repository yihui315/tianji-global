# TianJi Project Instructions

## Mission
TianJi is a premium AI destiny reading platform. Every change must improve one or more of:
- product clarity
- conversion
- design consistency
- result-page depth
- sharing and retention
- trust and privacy

## Non-negotiable rules
1. Do not remove or weaken:
   - pricing
   - FAQ
   - trust/disclaimer
   - privacy protections
   - premium upgrade paths
2. Do not expose birth date, birth time, or birth location in public share pages by default.
3. AI must explain structured outputs, not invent calculations.
4. Prefer single-language rendering with explicit language switch, not mirrored bilingual blocks.
5. Preserve premium dark visual style and design token consistency.
6. Never merge directly to main. Always work on a branch and prepare a PR summary.

## Product priorities
1. Result pages
2. Relationship graph
3. Fortune timeline
4. Share systems
5. Homepage polish
6. SEO/GEO content integration

## Required checks before claiming completion
- typecheck
- lint
- test
- build
- audit-routes
- audit-copy
- audit-share
- audit-upgrade

## Output requirements
Every upgrade PR must include:
- what changed
- why it changed
- screenshots or route list
- tests run
- risks or follow-ups
