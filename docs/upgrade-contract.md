# TianJi Upgrade Contract

## Purpose
This contract defines the minimum product and engineering expectations for TianJi upgrades.

## Product completion criteria

### Homepage
- Hero exists
- Results preview exists
- Trust section exists
- Pricing exists
- FAQ exists
- Final CTA exists

### Result Page
- Hero summary exists
- Structured chart section exists
- AI insight sections exist
- Applications layer exists
- Timeline exists
- Premium locked section exists
- Trust/disclaimer exists

### Relationship
- Dedicated route exists
- Five dimensions exist:
  - attraction
  - communication
  - conflict
  - rhythm
  - longTerm
- Pattern headline exists
- Current 30-day window exists
- Share system exists
- Public shares hide sensitive birth data by default

### Fortune Timeline
- Dedicated component exists
- Supports dimensions:
  - overall
  - career
  - wealth
  - relationship
- Event markers exist
- Share/download path exists

### Share system
- Summary card exists
- Graph card exists
- One-liner card exists
- Copy/share action exists
- Privacy-safe defaults exist

## Design system criteria
- Shared tokens are used
- CTA hierarchy is consistent
- Chart family is visually consistent
- No mirrored bilingual body copy

## Engineering criteria
- All required checks pass
- No broken routes
- No obvious duplicate dead components
- No silent regressions in critical paths
